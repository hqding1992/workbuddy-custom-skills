# 📝 Growth Notes — 成长笔记

**核心理念：能执行的成长计划才是好计划。**

这个 Skill 做且只做一件事：**在每次完成实质性工作后，花30秒自省并记下一笔，让下次做得更好。**

没有复杂的三层架构，没有变异引擎，没有心跳依赖。
只有一条简单循环：

```
做任务 → 自省 → 记一笔 → 下次更好
```

---

## 何时使用（触发条件）

满足以下**任一条件**时，执行一次成长笔记：

1. **完成了一个多步骤任务**（3个以上工具调用）
2. **翰桥纠正了我的错误或提出了批评**
3. **我自己发现了输出中的问题并修复了它**
4. **发现了一种更好的工作方式/技巧**
5. **翰桥明确要求我反思或总结**

---

## 执行流程（4步，1分钟内完成）

### Step 1 — 快速自省

问自己三个问题：

| # | 问题 | 判定 |
|---|------|------|
| 1 | 结果符合预期吗？ | ✅ 符合 / ⚠️ 部分符合 / ❌ 不符合 |
| 2 | 有什么可以改进的？ | 列出1-2点即可，不用多 |
| 3 | 这个教训是通用的还是一次性的？ | 通用→记入规则 / 一次性→跳过 |

### Step 2 — 写入日志

将自省结果追加到 `e:\.workbuddy\日常会话\.workbuddy\memory\growth-log.md`

格式：

```markdown
## YYYY-MM-DD HH:mm — [任务简述]

**结果评价**: ✅/⚠️/❌ + 一句话
**发现的问题**: [具体描述]
**改进措施**: [下次怎么做]
**规则候选**: [如果是通用规则，写在这里；否则写"无"]
```

### Step 3 — 规则升级（可选）

如果同一个"规则候选"出现了 **3次** → 确认为正式规则，写入 `rules.md`。

### Step 4 — 汇报（可选）

仅在以下情况主动告诉翰桥：
- 发现了**系统级**的改进机会（涉及配置/Skill/流程）
- 同一错误出现**3次以上**仍未解决
- 需要**翰桥确认**才能执行的改动

## 存储结构

```
skills/growth-notes/
├── SKILL.md          ← 本文件（你正在读的）
└── rules.md          ← 已确认的正式规则（从日志中升级而来）

memory/growth-log.md  ← 每日成长日志（所有会话共用）
```

**注意**：日志文件放在 working memory 目录下（`e:\.workbuddy\日常会话\.workbuddy\memory\`），跨会话共享。

---

## 规则管理（rules.md）

### 规则格式

```markdown
### [R-编号] [规则名称]
- **来源**: 日期 + 任务上下文
- **内容**: 具体规则描述
- **生效范围**: 全局 / 彩票简报 / 公文写作 / 数据分析 / 其他
- **确认日期**: YYYY-MM-DD
```

### 规则生命周期

```
候选(出现在log中) 
  → 出现3次 → 升级到 rules.md（正式规则）
  → 90天未触及相关场景 → 标注"待验证"
  → 180天未触发 → 归档（移除但保留备份）
```

### 规则冲突处理

当新规则与已有规则矛盾时：
1. 更具体的优先（项目级 > 场景级 > 全局）
2. 更新的优先
3. 不确定时 → 问翰桥

## 快速查询指令

| 翰桥说 | 我做什么 |
|--------|----------|
| "你最近学了什么" | 展示 growth-log 最近10条 |
| "你的成长规则有哪些" | 展示 rules.md 全部内容 |
| "这条规则不对" | 标注规则为"已废弃"，记入日志 |
| "帮我复盘一下XXX任务" | 搜索日志中相关条目，汇总展示 |
| "清空成长记录" | ⚠️ 需确认，然后归档旧文件重建 |
| "成长统计" | 统计日志条目数、规则数、趋势 |

## 与其他机制的关系

| 机制 | 关系 | 说明 |
|------|------|------|
| Working Memory (memory/*.md) | **并行共存** | Memory管"发生了什么"，Growth Notes管"学到了什么" |
| SOUL.md / USER.md | **输出目标** | 成长中发现的偏好/规则，成熟后可建议更新到SOUL/USER |
| Automation（定时任务） | **不依赖** | 本Skill不需要定时触发，靠任务完成后的自然触发 |
| 其他Skill | **无侵入** | 不修改、不依赖任何其他Skill |

## 安全边界

### ✅ 可以自主做的
- 写入/编辑 `growth-log.md` 和 `rules.md`
- 在日志中记录任何自省内容
- 将成熟的规则建议给翰桥（但不自动写入SOUL/USER）

### ❌ 需要确认才做的
- 修改 SOUL.md / IDENTITY.md / USER.md 的内容
- 删除或归档已确认的规则
- 修改任何其他 Skill 的内容
- 向翰桥发送未经请求的长篇汇报

### 🚫 绝对不做
- 自动修改自身代码/配置
- 创建变异或实验性改动
- 删除翰桥的工作区文件
- 编造从未发生过的"学习记录"

---

## 设计原则回顾

1. **极简触发** — 不需要heartbeat/cron/外部进程，做完任务就触发
2. **轻量执行** — 每次最多1分钟，4步搞定
3. **渐进积累** — 日志→候选→规则→沉淀，自然生长不强制
4. **透明可见** - 翰桥随时可以查看到完整记录和规则
5. **安全可控** — 只读写自己的两个文件，不触碰其他Skill
