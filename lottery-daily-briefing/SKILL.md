# 🎰 彩票新闻每日简报

> 彩票行业新闻专项采编与简报生成工具。每日自动从官方渠道抓取信息，经分类筛选后生成标准化的五大类新闻简报。

---

## 📌 适用场景

- **彩票中心日常简报**：为福彩/体彩系统工作人员提供每日行业信息速览
- **政策动态追踪**：自动监控财政部、民政部、体育总局等监管机构的最新政策
- **开奖与中奖汇总**：一站式汇总 9 大彩种开奖结果和全国重大中奖信息
- **活动与渠道监控**：持续追踪各地公益活动和渠道创新案例

### 🎯 简报五大信息分类

| 分类 | 内容范围 | 时间约束 | 必填字段 |
|------|---------|---------|---------|

## 📋 执行流程（7 个阶段）

### Phase 1：准备

1. **读取配置文件**：按顺序读取同目录下的参考文件
   - `source-whitelist.md` → 官方信源白名单与优先级
   - `search-strategy.md` → 固定栏目抓取 + 结构化搜索策略
   - `provincial-sites.md` → 省级官网URL库（30省+深圳，含SPA/不可用清单）
   - `format-template.md` → 简报输出格式模板
   - `active-events.md` → 进行中活动/渠道追踪表

2. **确定时间范围**：
   - 当前日期记为 TODAY
   - 时间范围 = 昨日 12:00 至 今日 12:00（24 小时窗口）
   - 例：TODAY = 2026-04-03 → 范围 = 2026-04-02 12:00 至 2026-04-03 12:00

3. **读取工作记忆**（如有）：
   - 读取 `{workspace}/.workbuddy/memory/MEMORY.md` 获取长期配置
   - 读取 `{workspace}/.workbuddy/memory/{TODAY}.md` 获取今日已执行记录

### Phase 2：信息采集

**📌 前置：用户补充入口**（如有）
> 如果用户提供了微信公众号文章链接（`mp.weixin.qq.com/s/xxxxx`），优先用 `web_fetch` 抓取全文内容，纳入信息采集。这是获取公众号独家内容最可靠的方式。

按 `search-strategy.md` 中定义的流程执行：

| 步骤 | 内容 | 次数 | 说明 |
|------|------|------|------|
| **Step 0** | 官方固定栏目抓取 | 8次并行 + 2次串行（cwl.gov.cn） | 政策/公告主力；江苏本省活动/渠道（含江苏福彩2栏目：地市动态+公益活动）。cwl.gov.cn 的0.9/0.10串行执行，间隔5-10秒 |
| **Step 1** | 综合搜索 | 3 次并行 | 全国政策 + 江苏南京本地 + 中奖/渠道 |
| **Step 2** | 开奖数据采集 | 体彩4次并行 + 福彩1次串行（延后执行，见 search-strategy.md 反爬策略） | 体彩4专区页(大乐透/7星彩/排列3-5/7位数) → 等待5-10秒 → 福彩开奖公告页（cwl.gov.cn串行） |
| **Step 2.5** | ⭐ 省级官网全覆盖抓取 | 44次分6批并行 + 1次API + 9次Playwright串行 | 每天全覆盖30省+深圳（体彩+福彩），当天发布当天抓到（含青海福彩新闻中心聚合页 web_fetch 直抓） |
| **Step 3** | 微信公众号搜索 | 已废止（默认跳过） | 搜狗4条必搜（南京/江苏）索引滞后至2020-2023，已废止；江苏本地由 Step 0 官网 + Step 3.4 覆盖 |
| **Step 3.4** | 搜索引擎补充 | 1 次 | 公众号时效性不足时，用搜索引擎补充活动/渠道 |
| **Step 3.5** | 微信全文抓取 | 按需 | 对7天内的公众号文章 web_fetch 全文 |
| **Step 4** | ⭐ 深度抓取（文章详情页，**硬门禁·不可跳过**） | **必须** | 下钻详情页取结构化字段（活动时间/地点/主办/内容/公益属性/参与方式、渠道案例说明/创新点/效果）。⚠️ 列表页仅返回标题+日期+URL，字段只在详情页。**硬门禁**：生成简报前必须校验已深抓详情页篇数——活动两节详版≥8、渠道两节详版≥4，不足则**简报不得生成**并报警。详情 URL 来源：① Playwright 日志 `_urls_YYYYMMDD.txt`；② web_fetch 对列表页 AI 提取（SPA 站 href 为空时）；③ http-only 站用 `scripts/fetch_details.js` 抓正文。深抓遇 429 用**指数退避**（1/2/4/8s）重试，批间退避 3s，禁固定长 sleep。07-09 / 07-14 两次因跳过此步被翰桥打回。 |
| **Step 5** | 交叉验证 | — | 对照信源白名单，排除不合规来源 |
| **Step 6** | 补充搜索 | 0-4 次 | 仅在结果不足时按需触发 |

> 📌 **搜索效率**：Step 0(11) + Step 1(3) + Step 2(5) + Step 2.5(44 web_fetch + 1 API + 9 Playwright = 54次) + Step 3(4) + Step 3.4(1) = 每次简报基础 **约78-88 次请求**。
> 📌 **cwl.gov.cn 全链路串行**：Step 0 第2批(2次串行) + Step 1 缓冲 + Step 2 福彩开奖页(1次串行) = 共3次 cwl.gov.cn 请求，全部间隔5-10秒。
> 📌 **核心分工**：活动+渠道信息以 **Step 0（江苏本地）+ Step 2.5（全国省级官网全覆盖）** 为主力，Step 3 搜索为补充。
> 📌 **全覆盖**：每天全覆盖全部30省官网，确保当天发布的活动/渠道信息当天抓取，无延迟。

### Phase 3：内容分类与筛选

按五大类严格分类，遵循以下规则：

#### 标注规则

| 状态 | 标注方式 | 要求 |
|------|---------|------|
| 新增内容 | 【新增】 | 发布时间和活动时间均在窗口内 |
| 预告内容 | 【预告】 | 发布时间在窗口内，但活动尚未开始（活动开始日期在未来） |
| 持续进行中的活动 | 【持续进行中（非今日新增）】 | 必须引用原文链接 |
| 持续运营中的渠道 | 【持续运营中（发布于X月X日）】 | 必须引用原文链接 |
| 无法确认的状态 | 标注"待核实" | 不虚构内容 |

#### 信息来源验证

- ✅ 所有信息必须来自 `source-whitelist.md` 中的官方渠道
- ✅ 每条信息必须标注：来源、发布时间、原始链接
- ✅ **中奖信息来源优先级**：Step 2 官方开奖页面（cwl.gov.cn / js-lottery.com）> 官方公告页搜索 > 主流媒体转载（必须标注原始官方出处）
- ✅ **中奖信息追溯规则**：如官方开奖页面未提供完整中奖详情（地区/单注奖金等），必须通过搜索 `"游戏名 第{期号}期 开奖"` 从 cwl.gov.cn 或 lottery.gov.cn 获取官方公告页，**禁止直接使用非白名单来源（如足彩网、特区彩票网等）作为来源标注**
- ⚠️ **信源降级声明**：当 cwl.gov.cn 官方开奖公告页因反爬返回403导致福彩开奖数据降级到第三方渠道时，必须在"最新开奖"大类开头添加统一声明：`⚠️ 中国福彩网开奖公告暂不可访问，本期福彩开奖数据来源于第三方渠道，以官方公告为准。`
- ❌ 禁止使用未经核实的第三方解读
- ❌ 某类别无内容时，如实填写"无内容"或"无新增内容"

#### 内容过滤规则（防污染）

对 Step 0 / Step 1 / Step 2.5 采集到的所有标题进行关键词过滤，**不包含以下任一关键词的标题直接排除**：

| 保留关键词（满足其一即可） | 示例 |
|--------------------------|------|
| 彩票、福彩、体彩、公益金 | "福彩公益金资助项目" |
| 开奖、派奖、投注、奖池 | "双色球奖池升至16亿" |
| 网点、渠道、站点、代销 | "江苏体彩培训进站点" |
| 责任彩票、公益、慈善 | "责任彩票宣传进社区" |
| 游戏规则、休市、兑奖 | "大乐透游戏规则调整" |

**强制排除规则**：标题含以下内容的**直接删除**，不得进入简报：
- 宏观经济指标（GDP、进出口、旅游收入、消费数据等）
- 与彩票无关的体育赛事结果（不含彩票赞助/公益属性的纯赛果）
- 社会新闻、娱乐新闻、政治新闻

#### 去重规则

生成简报前，对所有采集条目执行**标题相似度去重**：

1. **跨步骤去重**：Step 0（江苏官网）与 Step 2.5（省级官网）结果交叉比对
2. **相似度阈值**：标题相似度 ≥ 80%（如仅日期/地区不同，内容实质相同）视为重复
3. **保留策略**：重复条目只保留**发布时间最新**的一条，其余删除
4. **标注要求**：如一条为"新增"、一条为"持续中"，保留"新增"并合并链接

#### 分类边界规则（防止错分）

| 原分类 | 误判场景 | 正确分类 | 判断标准 |
|--------|---------|---------|---------|
| 官方解读 | 中体彩/中福彩发布的**活动预告/营销通知** | **各地活动** | 内容核心是"某活动即将开始/正在进行"→活动；内容是"政策含义/规则说明"→官方解读 |
| 官方解读 | 派奖公告、促销通知 | **各地活动** | 凡涉及具体派奖/赠票/促销的，一律进"各地活动" |
| 各地活动 | 渠道建设/网点招募/终端升级通知 | **渠道拓展** | 内容聚焦"网点怎么做/怎么改"→渠道；聚焦"彩民怎么参与"→活动 |
| 持续进行中 | 活动已过期（超过结束日期） | **删除或移入归档** | 每条"持续中"必须核实结束日期，已过期立即移除 |

### Phase 4：简报生成

按 `format-template.md` 的格式生成简报文件。

#### 文件规范

- **文件名**：`今日彩票新闻简报_YYYYMMDD.md`
- **保存路径**：`{workspace}/`（当前工作区根目录）
- **标题下方必须标注时间范围**：`**（YYYY年MM月DD日12:00至YYYY年MM月DD日12:00）**`

### Phase 5：验证检查（必须执行）

生成简报后，逐项检查：

```
[ ] 文件名格式正确：今日彩票新闻简报_YYYYMMDD.md
[ ] 标题下方标注了时间范围（昨日12:00 - 今日12:00）
[ ] 政策动态条目的发布日期在时间范围内
[ ] 各地活动新增条目在时间范围内
[ ] 渠道拓展新增条目在时间范围内
[ ] 持续进行中的活动已单独标注并附链接
[ ] 持续运营中的渠道已单独标注并附链接
[ ] 所有开奖结果注明了开奖日期，且开奖日期在时间范围内或为窗口内最近一期
[ ] ⚠️ 开奖时间校验：开奖时间在今日12:00之后的彩种不应出现在"最新开奖"中，应仅出现在"开奖预告"（如7位数21:00开奖但简报12:00截止，则上一期数据在最新开奖、本期在预告）
[ ] 所有中奖信息注明了中奖日期，且来源为官方渠道（非白名单来源已追溯原始出处并标注）
[ ] 无虚构内容、无示例数据、无模板数据
[ ] 无宏观经济/旅游收入/进出口等明显无关内容混入
[ ] 无重复条目（同一事件在多个分类或同一分类内重复出现）
[ ] "持续进行中"的活动已核实结束日期，已过期的已移除或标注
[ ] 活动预告/派奖公告未被误归入"官方解读"
[ ] active-events.md 的"最后更新"日期已更新为 TODAY
```

任何检查项未通过 → 修正后重新检查。

### Phase 6：追踪表更新

简报生成完毕后，更新 `active-events.md`：

1. 移除已结束的活动/渠道（移入"已结束归档区"，保留 30 天）
2. 添加新发现的进行中活动/渠道
3. 更新所有条目的"最后确认日期"
4. ⚠️ **无论活动/渠道是否有变动，都必须将文件头部的"最后更新"日期更新为 TODAY**

- 将执行摘要追加到 `{workspace}/.workbuddy/memory/{TODAY}.md`
- 如有需要，更新 `{workspace}/.workbuddy/memory/MEMORY.md`

## 📤 分发与发送

简报生成后，可根据需要通过以下方式分发：

### 1. 本地保存（默认）
- 简报文件自动保存至 `{workspace}/今日彩票新闻简报_YYYYMMDD.md`

### 2. IMA 知识库（使用稳定分发脚本，推荐）
- **使用 `scripts/ima_distribute.js`**（两步法稳定分发，2026-07-08 终极修复）：

## 🔧 抓取排错经验（Step 2.5）

### SPA 站点抓取失败：先查 URL 是否路由到正确栏目，而非直接改选择器
- **现象**：Playwright 能打开页面（title 正常），但 `fetch_news.js` 提取到 0 条新闻。
- **根因（高频）**：Vue/React SPA 用 hash 路由（如 `/#/article/channel/list/1-9`）或分页路径（如 `/tcxw/list/page.html`），根路径/旧路径不渲染目标栏目列表，导致容器为空。
- **正确排错顺序**：
  1. 先用浏览器/Playwright 确认目标栏目 URL（用户常能提供正确的栏目深链）
  2. 换 URL 重试——多数情况换对栏目 URL 即解决，**无需改 `fetch_news.js` 的 wait_selector**
  3. 仅当 URL 正确但仍 0 条时，才排查 `wait_selector` / 提取策略
- **实证案例（2026-07-06）**：河南福彩（`/#/article/channel/list/1-9` 抓到25条）、云南福彩（`/list/xwzx-2008880236425531393` 抓到10条）原以为需改脚本，实测换 URL 即解决；陕西体彩老地址 `/tcxw` 返回403反爬，换 `/tcxw/list/page.html` 后 web_fetch 直接成功。

### web_fetch 返回 403 / 连接失败：区分"反爬"与"URL失效"
- `403`（伴随 nginx/WAF 特征）：URL 存活，是反爬/UA 拦截 → 换 UA、跟随重定向，或用 Playwright 兜底
- `000`/`fetch failed`/连接超时：curl 复测若同样 `000` → 本环境出口 IP 被拒（非 URL 问题）；若用户本机也打不开 → **URL 确认真失效**，标注待补
